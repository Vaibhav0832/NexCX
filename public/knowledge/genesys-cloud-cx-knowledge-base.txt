Question Number: 1
Category: Platform Overview - Overview
Question: What is Genesys Cloud CX?
Keywords: Genesys Cloud CX, CCaaS, overview, contact center platform
Answer: Genesys Cloud CX is a cloud-native, all-in-one contact center as a service (CCaaS) platform that unifies voice and digital channel engagement, workforce engagement management, and AI-powered orchestration in a single application. It is built on a microservices architecture deployed on AWS and is delivered entirely as a subscription service, eliminating the need for customers to manage on-premises telephony or contact center infrastructure.
Related Topics: CCaaS, cloud-native architecture, omnichannel engagement
Difficulty: Beginner

Question Number: 2
Category: Platform Overview - Overview
Question: What does "experience orchestration" mean in the context of Genesys Cloud CX?
Keywords: experience orchestration, journey, customer experience
Answer: Experience orchestration refers to Genesys Cloud CX's approach of coordinating customer and employee interactions across every channel and touchpoint based on context, intent, and journey data, rather than treating each interaction as an isolated event. The platform uses AI and journey data to determine the best routing, content, and next action at each step of an interaction.
Related Topics: Predictive Engagement, journey management, AI-powered routing
Difficulty: Beginner

Question Number: 8

Question: What are the main subscription tiers available for Genesys Cloud CX?
Keywords: editions, tiers, CX1, CX2, CX3, CX4

Question Number: 11
Category: Platform Overview - Licensing
Question: What is the difference between a named license and a concurrent license?
Keywords: named license, concurrent license, licensing model
Answer: A named license is assigned to a specific individual and allows that person to access Genesys Cloud CX at any time, regardless of how often they actually log in. A concurrent license is shared across a pool of users but only supports a defined number of simultaneous logins at any given time. Concurrent licenses typically carry a price premium relative to named licenses to reflect the flexibility they provide.
Related Topics: Licensing models, hourly interacting license, user provisioning
Difficulty: Intermediate

Question Number: 12

Question: Can named and concurrent licensing models be mixed within the same organization?
Keywords: licensing models, mixing, named, concurrent
Answer: No. Genesys Cloud CX does not permit mixing named and concurrent licensing models within a single organization; an organization must standardize on one model. However, organizations can mix license tiers (for example, CX 2 and CX 2 Digital) as long as the underlying licensing model remains consistent.
Related Topics: Licensing models, organizations, edition mixing

Question Number: 20

Question: What is the difference between a core region and a satellite region?

Answer: A core region hosts the complete set of Genesys Cloud services, while a satellite region extends the Global Media Fabric (GMF) to provide localized media tier services — such as real-time voice and video processing — without hosting the full application stack. Satellite regions help reduce latency and support in-region recording storage for data residency requirements.

Answer: Genesys Cloud uses its Global Media Fabric, a network spanning core and satellite regions, to keep real-time media such as voice and video traffic as close as possible to the customer and agent. This minimizes round-trip latency and improves audio and video quality for geographically distributed users.

Question Number: 30
Category: Platform Overview - High Availability
Question: How does Genesys Cloud CX achieve high availability within a region?
Keywords: high availability, Availability Zones, active-active
Answer: Genesys Cloud CX runs services in an active/active/active configuration across a minimum of three AWS Availability Zones per region. Each Availability Zone has independent power, networking, and physical infrastructure, so that the failure of one zone does not bring down the service, since traffic is automatically handled by the remaining zones.
Related Topics: Availability Zones, AWS regions, fault tolerance
Difficulty: Advanced

Question Number: 31
Category: Platform Overview - High Availability
Question: What happens if a single node in a Genesys Cloud cache cluster fails?
Keywords: cache cluster, node failure, automatic failover
Answer: Genesys Cloud uses multiple cache clusters spread across Availability Zones, so if a single node in a cluster fails, the remaining nodes in the cluster take over that node's workload within seconds. If an entire cache cluster fails, it can be redeployed and will resume caching data from its source of record within minutes.
Related Topics: Caching, fault tolerance, Availability Zones
Difficulty: Architect

Question Number: 59
Category: User Management - Roles
Question: How are licenses connected to roles in Genesys Cloud?
Keywords: licensing, role-license relationship, cost association
